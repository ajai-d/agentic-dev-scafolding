## What I Want To Build

A simple 3-tier Task Tracker app with a Web frontend, REST API backend, and a local SQLite database file. Users can create, view, complete, and delete tasks. The goal is to demonstrate a clean separation between UI, service/API, and data layers.

## Done Looks Like

- Web UI supports create, list, complete, and delete task flows
- API provides CRUD endpoints with validation and clear error responses
- Task data persists in a local SQLite file between restarts
- Basic API tests pass
- Project runs locally with simple setup instructions
